# 技术文档的写作规范

> 本规范旨在统一中文技术文档的写作风格，提高文档的可读性和专业性。

## 一、文档体系

### 1.1 结构

软件手册是一部完整的书，建议采用下面的结构：

- **简介**（Introduction）：[必备] [文件] 提供对产品和文档本身的总体的、扼要的说明
- **快速上手**（Getting Started）：[可选] [文件] 如何最快速地使用产品
- **入门篇**（Basics）：[必备] [目录] 又称"使用篇"，提供初级的使用教程
  - **环境准备**（Prerequisite）：[必备] [文件] 软件使用需要满足的前置条件
  - **安装**（Installation）：[可选] [文件] 软件的安装方法
  - **设置**（Configuration）：[必备] [文件] 软件的设置
- **进阶篇**（Advanced)：[可选] [目录] 又称"开发篇"，提供中高级的开发教程
- **API**（Reference）：[可选] [目录|文件] 软件 API 的逐一介绍
- **FAQ**：[可选] [文件] 常见问题解答
- **附录**（Appendix）：[可选] [目录] 不属于教程本身、但对阅读教程有帮助的内容
  - **Glossary**：[可选] [文件] 名词解释
  - **Recipes**：[可选] [文件] 最佳实践
  - **Troubleshooting**：[可选] [文件] 故障处理
  - **ChangeLog**：[可选] [文件] 版本说明
  - **Feedback**：[可选] [文件] 反馈方式

**参考范例**：
- [Redux 手册](https://redux.js.org/introduction/getting-started)
- [Atom 手册](http://flight-manual.atom.io/)

### 1.2 文件名规范

| 规则 | 错误示例 | 正确示例 |
|------|---------|---------|
| 不得含有空格 | - | - |
| 必须使用半角字符，不得使用全角字符（中文不能用于文件名） | `名词解释.md` | `glossary.md` |
| 建议只使用小写字母（说明文件如 README、LICENSE 可大写） | `TroubleShooting.md` | `troubleshooting.md` |
| 多单词用半角连词线（`-`）分隔 | `advanced_usage.md` | `advanced-usage.md` |

## 二、标题

### 2.1 层级

标题分为四级：

- **一级标题**：文章的标题
- **二级标题**：文章主要部分的大标题
- **三级标题**：二级标题下面一级的小标题
- **四级标题**：三级标题下面某一方面的小标题

```markdown
# 一级标题

## 二级标题

### 三级标题

#### 四级标题
```

### 2.2 标题原则

#### 原则 1：层级不能跳跃
一级标题下，不能直接出现三级标题。

```markdown
❌ 错误示例：
# 一级标题

### 三级标题（缺少二级标题）
```

#### 原则 2：避免孤立编号
同级标题只有一个时，应省略该层级。

```markdown
❌ 结构一（不佳）：
## 二级标题 A

### 三级标题 A（只有一个三级标题，可省略）

## 二级标题 B

✅ 结构二（推荐）：
## 二级标题 A
（直接写内容，不使用三级标题）

## 二级标题 B
```

#### 原则 3：下级标题不重复上级标题

```markdown
❌ 错误示例：
## 概述

### 概述（与上级标题同名）

✅ 正确示例：
## 概述

### 概述详情
```

#### 原则 4：谨慎使用四级标题
尽量避免四级标题，保持层级简单。三级标题下的并列内容建议使用项目列表。

```markdown
❌ 结构一（复杂）：
### 三级标题

#### 四级标题 A

#### 四级标题 B

#### 四级标题 C

✅ 结构二（简洁）：
### 三级标题

**（1）A**

**（2）B**

**（3）C**
```

## 三、文本

### 3.1 字间距

#### 中英混排
全角中文字符与半角英文字符之间，应有一个半角空格。

```
❌ 本文介绍如何快速启动Windows系统。

✅ 本文介绍如何快速启动 Windows 系统。
```

#### 中文与数字
全角中文字符与半角阿拉伯数字之间，**有无空格均可，但必须统一**。

```
✅ 2011年5月15日，我订购了5台笔记本电脑与10台平板电脑。

✅ 2011 年 5 月 15 日，我订购了 5 台笔记本电脑与 10 台平板电脑。
```

半角的百分号视同阿拉伯数字。

```
✅ 今年我国经济增长率是6.5%。

✅ 今年我国经济增长率是 6.5%。
```

#### 英文单位
英文单位若不翻译，单位前的阿拉伯数字与单位符号之间应留出适当空隙。

```
例1：一部容量为 16 GB 的智能手机

例2：1 h = 60 min = 3,600 s
```

#### 标点与英文/数字
半角英文字符和半角阿拉伯数字，与全角标点符号之间不留空格。

```
❌ 他的电脑是 MacBook Air 。

✅ 他的电脑是 MacBook Air。
```

### 3.2 句子

#### 控制句长

| 字数 | 接受度 |
|------|--------|
| < 20 字 | 推荐 |
| 20～29 字 | 可以接受 |
| 30～39 字 | 语义必须明确 |
| > 40 字 | **不可接受** |

逗号分割的长句，总长度不应超过 100 字或正文 3 行。

```
❌ 本产品适用于从由一台服务器进行动作控制的单一节点结构到由多台服务器进行动作控制的并行处理程序结构等多种体系结构。

✅ 本产品适用于多种体系结构。无论是由一台服务器（单一节点结构），还是由多台服务器（并行处理结构）进行动作控制，均可以使用本产品。
```

#### 使用简单句
尽量使用简单句和并列句，避免使用复合句。

```
并列句：他昨天生病了，没有参加会议。

复合句：那个昨天生病的人没有参加会议。（避免）
```

#### 使用肯定句
同样意思，尽量使用肯定句，不使用否定句。

```
❌ 请确认没有接通装置的电源。

✅ 请确认装置的电源已关闭。
```

#### 避免双重否定

```
❌ 没有删除权限的用户，不能删除此文件。

✅ 用户必须拥有删除权限，才能删除此文件。
```

### 3.3 写作风格

#### 主动语态
尽量不使用被动语态，改为使用主动语态。

```
❌ 假如此软件尚未被安装，

✅ 假如尚未安装这个软件，
```

#### 正式语言
不使用非正式的语言风格。

```
❌ Lady Gaga 的演唱会真是酷毙了，从没看过这么给力的表演！！！

✅ 无法参加本次活动，我深感遗憾。
```

#### 现代汉语
不使用冷僻、生造或文言文的词语，使用现代汉语常用表达。

```
❌ 这是唯二的快速启动的方法。

✅ 这是仅有的两种快速启动的方法。
```

#### "的、地、得"用法

| 结构 | 示例 |
|------|------|
| 形容词 + 的 + 名词 | 她露出了**开心的**笑容。 |
| 副词 + 地 + 动词 | 她**开心地**笑了。 |
| 动词 + 得 + 副词 | 她笑得**很开心**。 |

#### 明确指代
使用代词（"其"、"该"、"此"、"这"等）时，必须明确指代内容，保证只有一个含义。

```
❌ 从管理系统可以监视中继系统和受其直接控制的分配系统。
（"其"指代不明）

✅ 从管理系统可以监视两个系统：中继系统和受中继系统直接控制的分配系统。
```

#### 形容词限制
名词前不要使用过多的形容词。

```
❌ 此设备的使用必须在接受过本公司举办的正式的设备培训的技师的指导下进行。

✅ 此设备必须在技师的指导下使用，且指导技师必须接受过由本公司举办的正式设备培训。
```

### 3.4 英文处理

#### 复数还原
英文原文使用复数形式时，翻译成中文应还原为单数。

```
英文：...information stored in random access memory (RAMs)...

中文：⋯⋯存储在随机存取存储器（RAM）里的信息⋯⋯
```

#### 外文缩写
外文缩写可以使用半角圆点（`.`）表示缩写。

```
U.S.A.
Apple, Inc.
```

#### 省略号转换
表示中文时，英文省略号（`...`）应改为中文省略号（`⋯⋯`）。

```
英文：5 minutes later...

中文：5 分钟过去了⋯⋯
```

#### 书名处理
英文书名或电影名改用中文表达时，双引号应改为书名号。

```
英文：He published an article entitled "The Future of the Aviation".

中文：他发表了一篇名为《航空业的未来》的文章。
```

#### 首次标注
第一次出现英文词汇时，在括号中给出中文标注。此后直接使用英文缩写。

```
IOC（International Olympic Committee，国际奥林匹克委员会）。这样定义后，便可以直接使用"IOC"。
```

#### 大小写规范
专有名词中每个词第一个字母均应大写，非专有名词不需要大写。

```
专有名词："American Association of Physicists in Medicine"（美国医学物理学家协会）

非专有名词："online transaction processing"（在线事务处理）
```

## 四、段落

### 4.1 段落原则

- 一个段落只能有一个主题，或一个中心句子
- 段落的中心句子放在段首，对全段内容进行概述
- 一个段落的长度**不能超过七行**，最佳段落长度**小于等于四行**
- 段落的句子语气要使用陈述和肯定语气，避免使用感叹语气
- 段落之间使用一个空行隔开
- 段落开头不要留出空白字符

### 4.2 引用规范

引用第三方内容时，应注明出处。

```
One man's constant is another man's variable. — Alan Perlis
```

全篇转载时，在全文开头显著位置注明作者和出处，并链接至原文。

```
本文转载自 WikiQuote
```

使用外部图片时，必须在图片下方或文末标明来源。

```
本文部分图片来自 Wikipedia
```

## 五、数值

### 5.1 半角数字

阿拉伯数字一律使用半角形式，不得使用全角形式。

```
❌ 这件商品的价格是１０００元。

✅ 这件商品的价格是 1000 元。
```

### 5.2 千分号

数值为千位以上，应添加千分号（半角逗号）。

```
XXX 公司的实收资本为 ￥1,258,000 人民币。
```

- 4 位数值：千分号可选（`1000` 或 `1,000` 均可）
- 4 位以上数值：**必须**添加千分号

### 5.3 货币

货币应为阿拉伯数字，并在数字前写出货币符号，或在数字后写出货币中文名称。

```
$1,000
1,000 美元
```

英文货币名称建议参考国际标准 [ISO 4217](https://en.wikipedia.org/wiki/ISO_4217)。

### 5.4 数值范围

表示数值范围时，用波浪线（`～`）或一字线（`—`）连接。带有单位或百分号时，两个数字建议都要加上单位或百分号。

```
132 kg～234 kg

67%～89%
```

### 5.5 变化程度表示法

#### 增加
- **增加到**：表示定量（过去为一，现在为二）
- **增加了**：表示增量（过去为一，现在为三）

```
增加到过去的两倍（过去为一，现在为二）

增加了两倍（过去为一，现在为三）
```

#### 减少
- **降低到**：表示定量（定额是一百，现在是八十）
- **降低了**：表示减量（原来是一百，现在是二十）

```
降低到百分之八十（定额是一百，现在是八十）

降低了百分之八十（原来是一百，现在是二十）
```

⚠️ **注意**：不能用"降低 N 倍"或"减少 N 倍"的表示法，要用"降低百分之几"或"减少百分之几"。

## 六、标点符号

### 6.1 基本原则

1. 中文语句的标点符号，均应该采取**全角符号**
2. 如果整句为英文，则该句使用英文/半角标点
3. 句号、问号、叹号、逗号、顿号、分号和冒号**不得出现在一行之首**
4. 点号（句号、逗号、顿号、分号、冒号）**不得出现在标题末尾**；标号（引号、括号、破折号、省略号、书名号、着重号、间隔号、叹号、问号）可以

### 6.2 句号

- 中文语句的结尾处应该用全角句号（`。`）
- 句子末尾用括号加注时，句号应在括号之外

```
❌ 关于文件的输出，请参照第 1.3 节（见第 26 页。）

✅ 关于文件的输出，请参照第 1.3 节（见第 26 页）。
```

### 6.3 逗号

- 逗号（`，`）表示句子内部的一般性停顿
- 注意避免"一逗到底"（整个段落除了结尾，全部停顿都使用逗号）

### 6.4 顿号

- 句子内部的并列词，应该用全角顿号（`、`）分隔，即使并列词是英语也是如此

```
❌ 我最欣赏的科技公司有 Google, Facebook, 腾讯, 阿里和百度等。

✅ 我最欣赏的科技公司有 Google、Facebook、腾讯、阿里和百度等。
```

- 英文句子中，并列词语之间使用半角逗号（`,`）分隔

```
Microsoft Office includes Word, Excel, PowerPoint, Outlook and other components.
```

- 中文句子内部的并列词，最后一个尽量使用"和"连接，使句子更连贯

```
✅ 我最欣赏的科技公司有 Google、Facebook、腾讯、阿里，以及百度等。

✅ 我最欣赏的科技公司有 Google、Facebook、腾讯、阿里和百度等。（更优）
```

### 6.5 分号

分号（`；`）表示复句内部并列分句之间的停顿。

### 6.6 引号

- 引用时，应该使用全角双引号（`" "`），注意前后双引号不同

```
许多人都认为客户服务的核心是"友好"和"专业"。
```

- 引号里面还要用引号时，外面一层用双引号，里面一层用单引号（`' '`）

```
鲍勃解释道："我要放音乐，可萨利说，'不行！'。"
```

### 6.7 括号

- 补充说明时，使用全角圆括号（`（）`），括号前后不加空格

```
请确认所有的连接（电缆和接插件）均安装牢固。
```

- 括号中英文名称对照：

| 符号 | 英文 | 中文 |
|------|------|------|
| `{ }` | braces 或 curly brackets | 大括号 |
| `[ ]` | square brackets 或 brackets | 方括号 |
| `< >` | angled brackets | 尖括号 |
| `( )` | parentheses | 圆括号 |

### 6.8 冒号

- 全角冒号（`：`）常用在需要解释的词语后边，引出解释和说明

```
请确认以下几项内容：时间、地点、活动名称和来宾数量。
```

- 表示时间时，应使用半角冒号（`:`）

```
早上 8:00
```

### 6.9 省略号

- 省略号（`⋯⋯`）表示语句未完或语气不连续
- 省略号占两个汉字空间、包含六个省略点，不要使用`。。。`或`...`等非标准形式
- **省略号不应与"等"一起使用**

```
❌ 我们为会餐准备了香蕉、苹果、梨…等各色水果。

✅ 我们为会餐准备了各色水果，有香蕉、苹果、梨⋯⋯

✅ 我们为会餐准备了香蕉、苹果、梨等各色水果。
```

### 6.10 感叹号

- 应使用平静的语气叙述，尽量避免使用感叹号（`！`）
- 不得多个感叹号连用（`！！`或`!!!`）

### 6.11 破折号

- 破折号`————`一般用于进一步解释
- 破折号应占两个汉字的位置。如果只占一个汉字位置，前后应留出一个半角空格

```
直觉————尽管它并不总是可靠的————告诉我，这事可能出了些问题。

直觉 —— 尽管它并不总是可靠的 —— 告诉我，这事可能出了些问题。
```

### 6.12 连接号

#### 直线连接号（`-`）
占一个半角字符位置，用于：
- 两个名词的复合
- 图表编号

```
氧化-还原反应

图 1-1
```

#### 波浪连接号（`～`）或一字号（`—`）
占一个全角字符位置，用于数值范围（日期、时间或数字）。

```
2009 年～2011 年
```

⚠️ 注意：波浪连接号前后两个值都建议加上单位。

也可用汉字"至"代替：

```
周围温度：-20 °C 至 -10 °C
```
